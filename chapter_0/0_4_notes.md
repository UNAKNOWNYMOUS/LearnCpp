# 0.4 -- Introduction to C++ development
- How C++ programs get developed:
  - 1. Define the problem to solve
  - 2. Design a solution
  - 3. Write a program that implements the solution
  - 4. Compile the program
  - 5. Link object files
  - 6. Test program
  - 7. Debug (going back to step 4, rinse and repeat)
## Step 1: Define the problem that you would like to solve
- This is the "what" step, where you figure out what problem you are intending to solve.
## Step 2: Determine how you are going to solve the problem
- This is the "how" step, where you determine how you are going to solve the problem you came up with in step 1.
- A bug is any kind of programming error that prevents the program from operating correctly.
- As an aside...:
- Debugging is the act of removing bugs.
## Step 3: Write the program
- The set of C++ instructions that we input into the text editor is called the program's source code (often shortened to just code).
- Code editor - editor that is designed for programming.
- Tip:
- For advanced readers:
- C++ does not have any requirements for naming files. However, the de-facto standard is to name the first/primary source file created for a program `main.cpp`.
- The filename (`main`) makes it easy to determine which is the primary source code file, and the `.cpp` extension indicates that the file is a C++ source code file.
- Best practice: Name the first/primary source code file in each program `main.cpp`. This makes it easy to determine which source code file is the primary one.
